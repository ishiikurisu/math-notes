Cálculo numérico

14/03/2019

Introdução

O que é cálculo numérico (CN)? É o conjunto de ferramentas ou métodos para resolver problemas matemáticos de forma aproximada. Geralmente cuida de problemas que não possuem solução exata (analítica). Geralmente usa máquinas para nos auxiliar.

Duas fases importantes no fluxo do cálculo numérico:

  • Modelagem
  • Resolução

A escolha do método mais eficiente precisa levar em conta:

  • Precisão desejada
  • Velocidade de convergência
  • Esforço computacional despendido

Ideias importantes:

  • Iteração: consiste de uma aproximação inicial e da repetição de um procedimento que se repete até concluir um teste de parada.
  • Aproximação local

Problemas numéricos trabalham com medidas, não com números exatos. Uma medida física é um intervalo, não um número. Os métodos aproximados busca uma aproximação do que seria o valor exato.

Conversão de bases numéricas

Conversões numéricas são bastantes utilizadas pois geralmente utilizamos os números na base decimal, mas outras bases são utilizadas para representar os números em máquinas, em especial a base binária.

Simbologia

Uma base numérica representa a quantidade de números possíveis para se representar um número. Na base decimal, usamos 0, 1, 2, 3, 4, 5,6 ,7 ,8, e 9. Na base binária, usamos 0 e 1.

Conversão de bases

A calculadora do Windows e o Excel são capazes de realizar estas conversões automaticamente.

  • Converta `159` de decimal para hexadecimal
  • Converta `C40F` de hexadecimal para decimal
  • Converta `127` de decimal para binário
  • Converta `11011010100110001` de binário para decimal
  • Converta `1100001011000101` de binário para hexadecimal
  • Converta `7F2D` de hexadecimal para binário
  • Converta `1110010001000` de binário para octal
  • Converta `67201` de octal para binário

Erro e representação

Números irracionais possuem diversas caracteristícas e a mais relevante agora é que eles não podem ser representados por frações de números inteiros. Desta forma, sua representação em uma base numérica é sempre infinita (ou seja, precisaríamos de memória infinita para representá-los). Isto é um problema para o CN, pois operações numéricas com estes números sempre trarão um erro associado.

Curiosidade: raízes pares de números primos sempre são números irracionais.

  • Encontre os seguintes somatórios em uma calculadora e em um computador: `S = sum_(i=1)^(30000) x_i` onde `x_i = 0,5` e `x_i = 0,11`.

Provavelmente, os resultados destas somas serão diferentes, já que as calculadoras e os computadores utilizam representações numéricas diferentes.

Representação numérica

Uma máquina computacional representa um número real no sistema de aritmética de ponto flutuante (APF), no qual um número `x` tem a seguinte estrutura:

`x = +- (0,d_1 d_2 d_3 d_4 ... d_t) * b ^ e`

onde `\{ d_i \}_(i=1)^(t)` é a chamada mantissa; `b`, a base; e `e`, o expoente. `d_1 != 0 AA x`. `e in (m, M)`, chamados de limitantes inferior `m` e superior `M`.

Para representar um número em ponto flutuante na base `b` com `t` dígitos (chamados significativos) e com limites dos expoentes `m` e `M`, usa-se a notação `F(b, t,m,M)`. Portanto, um número em APF `F(b, t,m,M)` será representado por

`+- 0,d_1 d_2 d_3 d_4 ... d_t * b ^ e`

onde `d_1 != 0` e `-m <= e <= M`

  • Represente em APF `(235,89)_10`. Resposta: `0,23589 * 10^3`
  • Represente em APF `(0,000875)_10`. Resposta: `0,875 * 10^(-3)`
  • Represente em APF e submeta aos arredondamento e truncamento `235,89`, considerando `t =3` e `b = 10`. Resposta: `0,236 * 10^3` com arredondamento, `0,235 * 10^3` com truncamento.
  • Represente em APF e submeta aos arredondamento e truncamento `12,76`, considerando `t =3` e `b = 10`. Resposta: `0,128 * 10^2` com arredondamento, `0,127 * 10^3` com truncamento.
  • Considere `x_1 = 0,35; x_2 = -5,172; x_3 = 0,0123`. Represente estes números no sistema `F(10,3,2,2)`. Utilize truncamento se necessário. Respostas: `x_1 = 0,350 * 10^0; x_2 = 0,53913 * 10^4; x_3 = 0,3 * 10^(-3)`. Note que os dois últimos números não são representáveis no sistema dado.
